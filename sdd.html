1. Pendahuluan

1.1 Tujuan Dokumen

Dokumen ini menjelaskan rancangan teknis AI Essay Preparedness Grader untuk platform ProdiPlan.id, meliputi struktur arsitektur, rancangan basis data, antarmuka pengguna, dan mekanisme fungsional sistem.

Tujuan dokumen ini adalah sebagai panduan bagi tim pengembang dalam proses implementasi, pengujian, dan pemeliharaan sistem.

1.2 Ruang Lingkup

AI Essay Preparedness Grader adalah fitur inovatif untuk platform web ProdiPlan.id yang menggunakan kecerdasan buatan dalam bentuk conversational agent untuk menganalisis esai jawaban siswa mengenai jurusan kuliah pilihan mereka.

Tujuan utama:

  • Mengetahui apakah siswa benar-benar siap menjalani hidup di jurusan tersebut
  • Mengurangi risiko salah pilih jurusan
  • Memberikan gambaran yang lebih realistis kepada calon mahasiswa

2. Deskripsi Umum Sistem

2.1 Perspektif Sistem

AI Essay Preparedness Grader dirancang sebagai fitur backend untuk platform ProdiPlan.id yang membantu siswa SMA dalam menentukan kesiapan mereka untuk jurusan kuliah tertentu melalui sesi tanya jawab interaktif dengan AI.

2.2 Fungsi Utama Sistem

  1. Autentikasi Pengguna: Registrasi, login, dan manajemen profil pengguna
  2. Manajemen Sesi Grading: Membuat dan mengelola sesi tanya jawab
  3. Conversational AI: Interaksi real-time dengan AI untuk evaluasi kesiapan
  4. Scoring & Analysis: Penilaian jawaban dan analisis komprehensif
  5. Laporan Hasil: Generasi laporan kesiapan dengan rekomendasi

2.3 Karakteristik Pengguna

Tipe Pengguna Karakteristik Kebutuhan
Siswa SMA Usia 15-18 tahun, calon mahasiswa Evaluasi kesiapan jurusan, rekomendasi
Admin Pengelola sistem Monitoring, manajemen pengguna

2.4 Arsitektur Sistem

Sistem menggunakan arsitektur microservices yang terdiri dari:

  • API Gateway (Entry point)
  • Auth Service (Autentikasi)
  • Session Service (Manajemen sesi)
  • Grading Worker (AI Processing)
  • Result Service (Hasil analisis)
  • WebSocket Service (Real-time communication)
  • Infrastructure (PostgreSQL, Redis)

4. Rancangan Basis Data

4.1 Struktur Penyimpanan Data (Entity Relationship Diagram)

Sistem menyimpan data dalam 4 tabel utama yang saling terhubung:

  • Tabel 1: USERS (Data Pengguna)
    Menyimpan informasi profile setiap siswa seperti nama, email, asal sekolah, dan jurusan impian mereka.
  • Tabel 2: GRADING_SESSIONS (Sesi Grading)
    Mencatat setiap sesi grading yang dibuat siswa.
  • Tabel 3: MESSAGES (Pesan/Percakapan)
    Menyimpan setiap pertanyaan dari AI dan jawaban dari siswa selama sesi.
  • Tabel 4: GRADING_RESULTS (Hasil Akhir)
    Menyimpan hasil final dari sesi grading.

4.2 Deskripsi Tabel

4.2.1 Tabel Users

Field Tipe Data Constraint Deskripsi
id UUID PK Primary key unik untuk setiap user
email VARCHAR(255) UNIQUE, NOT NULL Email unik user untuk login
full_name VARCHAR(255) NOT NULL Nama lengkap user
birth_date DATE NOT NULL Tanggal lahir user
school_origin VARCHAR(255) NOT NULL Asal sekolah user
dream_major VARCHAR(255) NOT NULL Jurusan impian user
firebase_uid VARCHAR(255) UNIQUE, NOT NULL UID dari Firebase Authentication

4.2.2 Tabel Grading Sessions

Field Tipe Data Constraint Deskripsi
id UUID PK Primary key unik untuk setiap sesi
user_id UUID FK → users.id Foreign key ke tabel users
status ENUM DEFAULT 'active' Status sesi (active, completed, expired)
current_score INTEGER DEFAULT 0 Skor akumulasi saat ini
threshold_score INTEGER DEFAULT 70 Threshold skor untuk menyelesaikan sesi

4.2.3 Tabel Messages

Field Tipe Data Constraint Deskripsi
id UUID PK Primary key unik untuk setiap pesan
session_id UUID FK → grading_sessions.id Foreign key ke tabel grading_sessions
message_type ENUM NOT NULL Tipe pesan (question/answer)
content TEXT NOT NULL Isi pesan
score INTEGER NULLABLE Skor untuk jawaban user (0-100)

4.2.4 Tabel Grading Results

Field Tipe Data Constraint Deskripsi
id UUID PK Primary key unik untuk setiap hasil
session_id UUID FK, UNIQUE Foreign key ke grading_sessions (1:1)
final_score INTEGER NOT NULL Skor akhir dari sesi (0-100)
readiness_level ENUM NOT NULL Level kesiapan user
analysis_report JSONB NOT NULL Laporan analisis komprehensif

5. Rancangan Antarmuka Pengguna

5.1 Alur Navigasi Sistem

Sistem memiliki alur: Landing Page → Login/Register → Dashboard → Grading Session → Result Page

Bagian A: Rancangan Front-End (Aplikasi Siswa)

Front-end adalah bagian aplikasi yang akan dilihat dan digunakan langsung oleh siswa SMA. Aplikasi ini dibangun menggunakan Next.js 14 dengan bahasa pemrograman TypeScript.

A.1.1 Teknologi yang Digunakan

Teknologi Fungsi
Next.js 14 Framework utama
TypeScript Bahasa pemrograman
Tailwind CSS Styling/tampilan
Framer Motion Animasi
React Query Manajemen data
Socket.io Client Komunikasi real-time

A.2 Rancangan Halaman-Halaman Utama

  • Landing Page: Halaman pertama yang dilihat pengunjung.
  • Halaman Autentikasi: Login dan Register.
  • Dashboard Pengguna: Ringkasan dan akses cepat ke fitur.
  • Halaman Essay Grader: Halaman utama untuk melakukan tes kesiapan.
  • Halaman Profil: Informasi profil dan riwayat assessment.
  • Halaman Hasil: Menampilkan hasil analisis AI secara detail.

A.2.1 Landing Page (Halaman Utama)

Tujuan: Halaman pertama yang dilihat pengunjung untuk memperkenalkan platform ProdiPlan dan mengajak mereka mendaftar.

Fitur Hero Section:

  • Judul utama: "Temukan Jurusan Kuliah yang Tepat untuk Anda dalam 5 Menit"
  • Animasi teks menggunakan SplitText
  • Tombol "Mulai Gratis" dan "Lihat Demo"

A.2.4 Halaman Essay Grader (Tes Kesiapan)

Tujuan: Halaman utama untuk melakukan tes kesiapan jurusan melalui serangkaian pertanyaan essay.

Fitur Real-time:

  • Koneksi WebSocket untuk menerima pertanyaan dari AI secara langsung
  • Update skor secara real-time setelah menjawab
  • Auto-submit jika waktu habis

Admin Panel adalah aplikasi terpisah yang digunakan oleh administrator untuk memantau dan mengelola sistem.

7. Rancangan Keamanan

7.1 Autentikasi

Sistem menggunakan Firebase Authentication sebagai identity provider utama.

  • User Registration: User mendaftar via Firebase, data profil di PostgreSQL.
  • JWT Token Management: Access token 1 jam, refresh token 7 hari.

7.2 Autorisasi

Role-Based Access Control (RBAC):

Role Permissions
User Create session, view own results, manage profile
Admin All user permissions + system monitoring + user management

7.3 Data Protection

  • Data in Transit: HTTPS/TLS via Cloudflare Tunnel, WSS untuk WebSocket.
  • Data at Rest: PostgreSQL encryption, Password hashing by Firebase.
  • Input Validation: Zod Schema Validation, SQL Injection Prevention.

7.4 Rate Limiting

Endpoint Rate Limit Window
/auth/login 5 requests 15 minutes
/grading-sessions 10 requests 1 hour

8. Rancangan Integrasi dan Deployment

8.1 Containerization

Sistem di-deploy menggunakan Docker Compose dengan services: cloudflared, postgres, redis, auth-service, api-gateway, session-service, grading-worker, websocket-service, dan result-service.

8.4 Deployment Strategy

Cloudflare Tunnel Configuration:

  • Domain: prodiplan.my.id
  • Tunnel routes to: api-gateway:4000
  • Zero Trust security enabled
